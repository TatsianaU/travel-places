# Домашнее задание

## Задание 1. Кнопка «Отмена» на странице создания места

На странице создания места (`/places/new`) сейчас есть ссылка «Назад к списку» сверху.

1. Добавьте в нижнюю часть страницы (рядом с кнопками формы или под ней) понятную ссылку **«Отмена»**, ведущую обратно на список `/places`.
2. Используйте для неё компонент `Link` (это переход по клику пользователя, а не из кода).

Проверьте: со страницы создания по ссылке «Отмена» вы возвращаетесь к списку без перезагрузки страницы.

## Задание 2. Ссылка «Редактировать» на странице деталей места

Откройте страницу деталей места (`/places/:id`).

1. Убедитесь, что на ней есть ссылка «Редактировать», ведущая на `/places/:id/edit`.
2. Если хотите — оформите её аккуратнее через обычный CSS (например, как заметную кнопку).

Проверьте: со страницы деталей по ссылке «Редактировать» открывается форма редактирования именно этого места, с уже заполненными полями.

## Задание 3. Возврат на список после удаления со страницы деталей (необязательное)

Сейчас удаление работает на странице списка. Добавьте возможность удалить место **со страницы деталей** и после удаления автоматически вернуться к списку.

1. На странице деталей (`/places/:id`) добавьте кнопку «Удалить».
2. По нажатию вызовите удаление места через существующую функцию `deletePlace(id)` из `src/api/places.ts`.
3. После успешного удаления выполните программный переход на список с помощью `useNavigate`:

   ```ts
   navigate('/places')
   ```

Подсказки:

- удаление меняет данные на сервере — это действие, а переход после него делается из кода, поэтому здесь нужен именно `useNavigate`, а не `Link`;
- по желанию можно перед удалением показать подтверждение через существующий компонент `ConfirmModal` — но это не обязательно;
- не забудьте обработать возможную ошибку запроса (например, показать сообщение).

Проверьте: на странице деталей нажали «Удалить» → место удалилось на сервере → вы оказались на странице списка, и удалённого места там больше нет.
