# Домашнее задание

## Задание 1. Страница «О проекте»

Добавьте новую страницу «О проекте».

1. Создайте компонент `src/pages/AboutPage.tsx` с коротким текстом о приложении (2–3 предложения).
2. Добавьте маршрут `/about` в `App.tsx`.
3. Добавьте пункт «О проекте» в навигацию в `Header` через `NavLink`.

Проверьте, что при клике на пункт меню страница меняется без перезагрузки, а активный пункт подсвечивается.

## Задание 2. Кнопка «На главную» на странице 404

Дополните страницу 404 (`NotFoundPage`).

1. Убедитесь, что на странице есть понятный текст на русском языке.
2. Добавьте на неё заметную ссылку-кнопку возврата на главную страницу через `Link` (если её ещё нет — оформите красиво через обычный CSS).

Проверьте: введите в адресной строке любой несуществующий адрес (например, `/qwerty`) и убедитесь, что открывается ваша страница 404 со ссылкой назад.

## Задание 3. Улучшите страницу деталей места (необязательное)

Доработайте страницу деталей (`PlaceDetailsPage`).

1. Выведите рядом с названием места небольшой блок, который показывает id места из URL (вы получаете его через `useParams`). Подпишите его, например: «Идентификатор места: 3».
2. Аккуратно обработайте ситуацию, когда место не найдено: если пользователь открыл `/places/9999`, он должен увидеть понятное сообщение на русском и ссылку «Назад к списку». (Базовая обработка уже есть — убедитесь, что текст понятный, и при желании оформите блок через CSS.)
3. Проверьте поведение вручную:
   - откройте существующее место (например, `/places/2`) — данные загружаются;
   - откройте несуществующее (`/places/9999`) — показывается сообщение и ссылка назад;
   - вернитесь к списку по ссылке и убедитесь, что переход прошёл без перезагрузки страницы.

> Подсказка: значение из `useParams` всегда приходит **строкой**. Для вывода id на экран преобразовывать ничего не нужно.
