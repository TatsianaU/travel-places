# Guide

## Сортировка «Новые сначала» (`-visitedYear`)

На странице `/places` доступен режим сортировки «Новые сначала» — по полю `visitedYear` по убыванию.

- В URL это значение хранится как `?sort=-visitedYear`.
- Сортировка выполняется на сервере: API-слой (`fetchPlacesPage`) кладёт значение в `_sort`, а префикс `-` json-server v1 трактует как порядок по убыванию (`desc`).
- Отдельный параметр `order` не вводится — направление «зашито» в само значение `sort`.

### Поведение с пустыми `visitedYear`

Если у места нет `visitedYear` (например, статусы `wishlist` / `planned` без года), json-server отправляет такие записи **в конец списка**. Это штатное поведение сортировки по `undefined`-значениям, отдельной обработки не требуется.
