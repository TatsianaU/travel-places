# Домашнее задание

## Задание 1. Фильтр «Только избранное» через URL

Добавьте в панель фильтров на странице `/places` дополнительный фильтр «Только избранное» — переключатель (чекбокс), который показывает только те места, что лежат в избранном (`wishlistIds` в `localStorage`).

1. Заведите в URL отдельный параметр, например `?favorites=1`.
2. Прочитайте его через `useSearchParams.get('favorites')`. Если значение есть и не пустое — фильтр активен, иначе нет.
3. На каждое переключение чекбокса обновляйте URL через тот же хелпер `updateParam`, что и для `status` / `sort` / `view` (помните: пустое значение — удаляем параметр, чтобы не оставлять `?favorites=` в адресе).
4. В конвейере фильтрации после фильтра по статусу добавьте фильтр по избранному: оставлять только места, чьи id есть в `wishlistIds`.
5. Не забудьте включить `favorites` в признак «есть ли активные фильтры», чтобы кнопка «Сбросить» появлялась и сбрасывала его тоже.

Проверьте: включили чекбокс — в URL появилось `?favorites=1`, в списке остались только избранные места. Скопировали ссылку — открыли в новой вкладке — фильтр восстановился.

## Задание 2. Показ активных фильтров

Добавьте на странице `/places` небольшой блок «Активные фильтры», который выводит, какие именно фильтры сейчас применены.

Например:

> Активные фильтры: поиск «italy», статус «Посещено», сортировка «По названию», вид «Таблица»

1. Покажите блок только если есть хотя бы один активный фильтр.
2. Аккуратно подписывайте значения (для статуса используйте человекочитаемые подписи из `PLACE_STATUS_LABEL`, а не сырое `visited`). Для вида — «Карточки» / «Таблица».
3. Не дублируйте логику чтения — берите значения из тех же переменных `search`, `status`, `sort`, `view`, что уже есть на странице.
4. Сделайте вид (`view`) видимым в блоке, только когда он отличается от значения по умолчанию (`cards`) — иначе блок будет всегда непустым.

Проверьте: блок появляется и пропадает в зависимости от наличия фильтров; подписи на русском, без сырых ключей.

## Задание 3. Сохранять «вид» при сбросе фильтров (необязательное, сложное)

Сейчас кнопка «Сбросить фильтры» очищает всё, включая режим отображения. Это удобно для объяснения, но для пользователя бывает раздражающе: он работал в таблице, нажал «Сбросить» — и его выкинуло обратно на карточки.

Доработайте поведение так, чтобы при сбросе фильтров `view` **оставался текущим**:

1. Перестройте `handleResetFilters`: вместо `setSearchParams({})` соберите новый `URLSearchParams` руками. Если текущий `view !== 'cards'` — добавьте `view` в новые параметры, остальные не добавляйте.
2. Подсказка: пустой URLSearchParams можно создать как `new URLSearchParams()` и затем `.set('view', view)`.
3. Убедитесь, что в признаке «есть ли активные фильтры» (`hasActiveFilters`) `view` по-прежнему учитывается отдельно — иначе после сброса кнопка «Сбросить» может остаться видимой, если view = 'table' (это нормально: пользователь видит, что есть «лишний» URL-параметр и может убрать его повторным сбросом, ИЛИ — вариант для усложнения — сделайте отдельную кнопку «Сбросить вид»).
4. Проверьте сценарий: выставили статус = «Посещено», сортировку = «По названию», вид = «Таблица», нажали «Сбросить» → URL стал `/places?view=table`, список таблицей. Без активных фильтров кнопка «Сбросить» не показывается.

Подсказки:

- решите для себя одно правило поведения и кратко зафиксируйте его в комментарии у `handleResetFilters` — преподаватель будет смотреть именно туда;
- если задание кажется слишком объёмным, остановитесь на варианте «при сбросе `view` сохраняется», без отдельной кнопки «Сбросить вид»;
- ещё одна полезная мини-задача (по желанию): добавьте обработку случая, когда пользователь руками вписал `?view=spaceship`. Проверьте, что в таблице это не ломает страницу, а вид трактуется как «карточки».
